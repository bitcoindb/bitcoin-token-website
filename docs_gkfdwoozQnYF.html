BitcoinToken Docs

VERSION 0.1

BitcoinToken consists of three tools: a Bitcoin Cash wallet, a database for reading and writing to the blockchain, and a token solution. The tools are implemented in Javascript and run in the browser and in node.

The Tech Stack

BitcoinToken has a layered architecture, where lower levels provide functionality for the levels above. Integrating BitcoinToken into your app is as easy as requiring the npm package.

Our software is not available on npm yet. Email clemens@bitcointoken.com to obtain an early preview of the library

Your app

Make use of the functionality provided by the layers below

BitcoinToken

Create tokens and smart contracts on Bitcoin Cash

BitcoinDb

Store, update, and retrieve json data on the Bitcoin Cash blockchain.

BitcoinWallet

Receive, hold, and send Bitcoin Cash.

BitcoinSource

A high quality Javascript implementation of Bitcoin Cash. Click to see source code on Github.

The BitcoinToken suite consists of three classes, all of which are exposed at the root level. 

              const Bitcoin = require('./bitcoin-token')
  
              const BitcoinWallet = Bitcoin.Wallet
              const BitcoinDb = Bitcoin.BitcoinDb
              const BitcoinToken = Bitcoin.Token

BitcoinWallet

BitcoinWallet is a Bitcoin wallet that works out of the box. It allows users to store, send, and receive Bitcoin Cash. Basic usage is as follows. 

                const wallet = new BitcoinWallet()
                await wallet.getAddress()                  
                await wallet.getBalance()                  
                await wallet.send(1, <public key>)

Initialization

There are two ways to initialize a BitcoinWallet. To create a wallet and initialize it with a fresh random master private key, just call the constructor. 

                const wallet = new BitcoinWallet()

To initialize the wallet from an existing private key call fromHdPrivateKey 

                const masterPrivateKey = 'tprv8ZgxMBicQKsPefX1...'
                const wallet = BitcoinWallet.fromHdPrivateKey(masterPrivateKey)

Receiving Bitcoin Cash

You wallet comes with over 2 billion accounts, numbered from 0 to 2.147.483.648. You can access these accounts by executing the following commands. The default account is 0. 

                const accountNumber = <some number between 0 and 2.147.483.648>
                const privateKey = wallet.getPrivateKey(accountNumber)
                const publicKey = wallet.getPublicKey(accountNumber)
                const address = wallet.getAddress(accountNumber)

To fund the wallet get a deposit address and fund through a testnet faucet (eg https://coinfaucet.eu/en/bch-testnet/).

Sending Bitcoin Cash

After you have funded your wallet, you can send Bitcoin Cash to another address as follows. 

                const amount = <number of satoshi to be sent>
                const address = <some Bitcoin Cash address>
                const res = wallet.send(amount, address)

By default, the wallet will send funds from account number 0. If you want to send from another account, plug in that accounts number as the third parameter 

                const accountNumber = <an account number between 0 and 2.147.483.648>
                const res = wallet.send(amount, address, accountNumber)

Checking the balance

You can check the balance of an account by calling getBalance with the account number. Like everywhere else the account number can be ommited and defaults to 0. 

                const balance = wallet.getBalance(accountNumber)

BitcoinDb

BitcoinBd lets you store, read, and update json files on the blockchain. Every json file stored on the blockchain has an owner and only the owner can update the json. We are currently only supporting json objects with string values of less than 350 charachters. In the future we will add support for arbitrary json data.

Basic usage is as follows. 

                  const db = new BitcoinDb()
                  const id = await db.put({ data: { hello: 'Bitcoin' }})
                  const res = await db.get(id)    // res will be { data: { hello: 'Bitcoin' }}

Initialization

There are three ways to initialize a BitcoinWallet: To create a wallet and initialize it with a fresh random master private key, just call the constructor. To initialize from an existing private key call fromHdPrivateKey. The third option is to pass in a BitcoinWallet object 

                const db1 = new BitcoinDb()
  
                const masterPrivateKey = 'tprv8ZgxMBicQKsPefX1...'
                const db2 = BitcoinDb.fromHdPrivateKey(masterPrivateKey)
  
                const bitcoinWallet = new BitcoinWallet()
                const db3 = new BitcoinDb(bitcoinWallet)

Storing data

To store some data in json format create and object containing that data as a value under the key data. 

                const data = { a: 'aaa', b: 'bbb' }
                await db1.put({ data })

Anyone can read the data that is stored in BitcoinDB, however only the owner can update the data. By default the current user will be the owner of newly created data. To designate another owner, pass in that users public key 

                const data = { a: 'aaa', b: 'bbb' }
                const owners = [ db2.getPublicKey() ]
                await db1.put({ data, owners })

The return value of db.put is a json object that contains the transaction id as well as the output number of the output encoding the data. 

                  {
                    txId: 'f9ed485e2a528cf1c0570b8f1c65b1521a68b1e63b50f3011e9e7d4d2eeecf04',
                    outputNumber: 0
                  }

Internally, the command db.put builds and broadcasts a Bitcoin Cash transaction with one output that contains a script encoding the data. The owner is the user that has the private key that can spend (and hence update) that data.

Reading data

The id returned from db.put can be used to lookup the json object stored on the blockchain. You can pass that identifier into db.get to retrieve the json object. 

                const id = await db.put({ data: { c: 'ccc' } })
                await db.get(id)

The value returned from db.get is a json object with three keys: data contains the initially stored data, owners is the list of the public keys of the owners, and amount contains the number of satoshi contained in the output that stores the data. 

                { 
                    data: { c: 'ccc' },
                    owners: [ <PublicKey: 03fc00d6e3044a17b17ff443f14f2eb23181071c9140a71a7cdc26fcc5d6d597e8> ],
                    amount: 5000
                }

Updating data

You can use db.update to update data stored with BitcoinDb. db.update expects two lists as arguments: a list of output ids (as returned from db.put) that indicates which data to update, and a list of the same length that specifies what to update the data to. This list contains entries that are json objects with two keys, data and owners, just like the objects passed to db.put. 

                const id1 = await db.put({ data: { c: 'ccc' } })
                const id2 = await db.update([id1], [{ data: { d: 'ddd' } }])

Internally, db.update builds and broadcasts a Bitcoin Cash transaction that spends the output id1 and spends them into a new output at id2 containing the new data. It is important to note that id1 and id2 are different outputs. To read the updated data you need to call db.get(id2).

BitcoinToken

Bitcoin token lets you create tokens and other smart contracts on top of Bitcoin Cash. The basic usage is as follows. We currently support fungible tokens, similar to Ethereums ERC20 contract. We will be adding support for more smart contracts soon. 

                  const token = new BitcoinToken()
                  await token.create({ data: { balance: '10' }})
                  await token.getBalance()                  
                  await token.send(1, <public key>)

Initialization

Like BitcoinDb, BitcoinToken can be initialized in three different ways: To create a token from a random master private key, call the constructor. Or, initialize from an existing private key using fromHdPrivateKey. The third option is to pass in a BitcoinDb object 

                const token1 = new BitcoinToken()
  
                const masterPrivateKey = 'tprv8ZgxMBicQKsPefX1...'
                const token2 = BitcoinToken.fromHdPrivateKey(masterPrivateKey)
  
                const db = new BitcoinDb()
                const token3 = new BitcoinToken(db)

Creating (issuing) a Token

You can issue a token by calling token.create. To parameter must be the same kind of json object that is passed into db.put. You need to initialize the balance (the number of tokens existing at issuance) to some positive number 

                await token.create({ data: { balance: '10' }})

Checking the token balance

You can check how many tokens you currently own by calling token.getBalance. The return value is a number. 

                await token.getBalance()
