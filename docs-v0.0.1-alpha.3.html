BitcoinToken Docs

VERSION v0.0.1-alpha.3

BitcoinToken is a toolkit for building blockchain enabled web applications on top of Bitcoin Cash. It consists of three tools: a wallet, a database built on the blockchain, and a token solution. These tools can be combined in many different way to adapt to the desired application. The code is written in Javascript and runs in the browser and in node.

The Tech Stack

BitcoinToken has a layered architecture, where lower levels provide functionality for the levels above.

Your app

Your application can use of the functionality provided by the layers below

BitcoinToken

Create tokens and smart contracts

BitcoinDb

Store, update, and retrieve json data on the blockchain.

BitcoinWallet

Receive, hold, and send Bitcoin Cash.

BitcoinSource

A community driven initiative to produce a readable implementation of Bitcoin Cash in Javascript.

Obtaining the code

Our source code is not publically available yet. If would like to obtain an early version to test, email clemens@bitcointoken.com.

The BitcoinToken package exposes the three classes at the root level.

        const Bitcoin = require('./bitcoin-token')

        const BitcoinWallet = Bitcoin.Wallet
        const BitcoinDb = Bitcoin.Db
        const BitcoinToken = Bitcoin.Token

BitcoinWallet

BitcoinWallet is a HD Javascript Bitcoin Cash wallet. It can store, send and receive bitcoin, and generate private keys as user accounts. Every Bitcoin wallet can deterministically generate more secure BitcoinWallets, which makes it possible to build trees of wallets that can adapt to the use case.

          const wallet = new BitcoinWallet()
          await wallet.getAddress()                  
          await wallet.getBalance()                  
          await wallet.send(1, <public key>)
          const childWallet3 = wallet.derive(3)

Initialization

To create a wallet from random, call the constructor. To initialize the wallet from an existing private key call BitcoinWallet.fromHdPrivateKey() with a string encoding an extended private key as specified in BIP32. BitcoinWallet can securely generate such a key for you when you call BitcoinWallet.getHdPrivateKey().

          const randomWallet = new BitcoinWallet()

          const hdPrivateKey = BitcoinWallet.getHdPrivateKey()
          const walletFromKey = BitcoinWallet.fromHdPrivateKey(hdPrivateKey)

Receiving Bitcoin Cash

To fund your wallet call wallet.getAddress() to obtain a Bitcoin Cash testnet deposit address. Then, paste that address into a testnet faucet to obtain free testnet coins.

The private and public key of a wallet are exposed via wallet.getPublicKey() and wallet.getPrivateKey().

          const address = wallet.getAddress()

          const publicKey = wallet.getPublicKey()
          const privateKey = wallet.getPrivateKey()

Sending Bitcoin Cash

After you have funded your wallet, you can send Bitcoin Cash to another address by calling wallet.send(). The return value is the id of the transaction that was broadcast.

              const txId = await wallet.send(amount, address)

Checking the balance

You can check the balance of the wallet by calling wallet.getBalance().

          const balance = await wallet.getBalance()

Deriving child wallets

You can securely derive 231 child BitcoinWallets from a BitcoinWallet. These wallets can derive further child wallets ad infinitum. As the process is deterministic, only one root HD private key must be stored, all wallets can be computed from that key.

          const childWallet = wallet.derive(3)
          const grandChildWallet = childWallet.derive(7)

BitcoinDb

BitcoinBd lets you store, read, and update json files on the blockchain. Reading from BitcoinDb is free and as fast as reading from any other database. Writing to BitcoinDb is associated with a one-time fee of about $100 per mb, payed in Bitcoin Cash. All data is stored in the Bitcoin Cash blockchain and is thus available globally 100% of the time. The data will be stored indefinitely at no recurring cost. In addition, BitcoinDb syncs in seconds across the globe.

          const db = new BitcoinDb()
          const id1 = await db.put({ a: 'aaa' })
          const data = await db.get(id1)
          const id2 = await db.update(id1, { b: 'bbb' })

Initialization

To create a wallet from random, call the constructor. To initialize from an existing private key call BitcoinDb.fromHdPrivateKey(). The third option is to pass in a BitcoinWallet object.

          const db1 = new BitcoinDb()

          const hdPrivateKey = BitcoinDb.getHdPrivateKey()
          const db2 = BitcoinDb.fromHdPrivateKey(hdPrivateKey)

          const bitcoinWallet = new BitcoinWallet()
          const db3 = new BitcoinDb(bitcoinWallet)

Storing data

The function db.put() takes a json object as input and encodes it in an output script of a Bitcoin Cash transaction. It then signs and broadcasts the transaction using Bitcoinwallet. We currently support json data with string keys and values that need to be shorter than 350 charachters. 

              await db1.put({ a: 'aaa' })

Data stored using BitcoinDb is stored on the blockchain and readable by anybody. If you want to restrict read access we recommend you encrypt data before you store it in BitcoinDb.

While anyone can read the data that is stored in BitcoinDB, only a specified user (group) called "owners" can update the data. To specify that group of users, you can pass an optional second parameter to db.put() that contains the public keys of the owners. By default the current user is the only owner.

          await db1.put({ a: 'aaa' }, [db2.getPublicKey()])

db.put() takes a third optional argument that specifies an account of satoshis that will be stored in one atomic unit together with that data and the owners. When you call db.put() this amount of satoshis that will be transferred from the user calling the function to the new owners of the data. This feature can be used to build a token exchange and has many other use cases.

          await db1.put({ a: 'aaa' }, [db2.getPublicKey()], 100000)

Reading data

The return value of db.put() is a json object that encode the location on the blockchain where the data is stored. It consists of a transaction id as well as the output number of the output encoding the data.

          { txId: 'f9ed485e2a528cf1c0570b8f1c65b1521a68b1e63b50f3011e9e7d4d2eeecf04', outputNumber: 0 }

The id returned from db.put() can be used to lookup the data stored in that location on the blockchain. BitcoinDb syncs across the globe in seconds, so data stored by one user is visible by every other user within seconds. To read data, pass the identifier returned by db.put() into db.get().

              const id = await db1.put({ a: 'aaa' })
              await db2.get(id)

The value returned from db.get() is a json object with three keys: data contains the initially stored data, owners is the list of the public keys of the owners, and amount contains the number of satoshi contained in the output that stores the data.

          { 
              data: { a: 'aaa' },
              owners: [ <PublicKey: 03fc00d6e3044a17b17ff443f14f2eb23181071c9140a71a7cdc26fcc5d6d597e8> ],
              amount: 5000
          }

Updating data

You can use db.update() to update data stored with BitcoinDb. The first parameter is the id of the output that contains the data that is to be updated. The second parameter is the new data. A third optional parameter can contain a list of owners like in db.put() that defaults to the current user. The fourth optional parameter is an amount of satoshis, also like in db.put(). 

          const id1 = await db1.put({ a: 'aaa' })
          const id2 = await db1.update(id1, { b: 'bbb' })
          const id3 = await db1.update(id2, { c: 'ccc' }, [db2.getPublicKey()])
          // the next line throws an error bc id3 is owned by db2
          const id4 = await db1.update(id3, { d: 'ddd' }, [db1.getPublicKey()], 100000)

Under the hood

Internally, db.update() builds and broadcasts the following Bitcoin Cash transaction: it spends the outputs specified by the first parameter and spends them into a new output that encodes the data from the second parameter. The output is an n-of-n multisig output, spendable by the owners from the third parameter. This output contains the number of Bitcoin specified by the last parameter.

BitcoinToken

Bitcoin token lets you create tokens and other smart contracts on top of Bitcoin Cash. We currently support fungible tokens, similar to Ethereums ERC20 contract. We will be adding support for more smart contracts soon.

          const token = new BitcoinToken()
          await token.create({ balance: '10' })
          await token.send(1, <public key>)
          await token.getBalance() // returns 9

Initialization

A BitcoinToken object can be created from random by calling the constructor. Or, it can be initialized from an existing private key using BitcoinToken.fromHdPrivateKey(). You can also pass an existing BitcoinDb object into the constructor.

          const token1 = new BitcoinToken()

          const hdPrivateKey = BitcoinDb.getHdPrivateKey()
          const token2 = BitcoinToken.fromHdPrivateKey(hdPrivateKey)

          const db = new BitcoinDb()
          const token3 = new BitcoinToken(db)

Issuing a Token

A token can be issued by calling token.create(). The parameter must be a json object with one key "balance" and a string value that encodes an integer. The balance encodes the numbers of tokens to be created. Initially all tokens are owned by the private key of the token object on which token.create() is called.

            await token.create({ balance: '10' })

Internally, token.create() calls the db.put() function of a BitcoinDb object that is a property of the token object. Then db.put() builds, signs and broadcasts a transaction using BitcoinWallet. That transaction stores the meta data of the token in the blockchain.

You can add your own meta data to the token creation transaction:

            await token.create({
                balance: '10',
                name: 'my-token',
                url: 'www.mytoken.com',
                ...
            })

Checking the token balance

The function token.getBalance() returns the number of tokens owned by the object.

              await token.getBalance()
