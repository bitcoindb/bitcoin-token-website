BitcoinToken

Build tokens and smart contracts in Bitcoin Cash.

BitcoinToken is a suite of tools to create tokens and smart contracts on top of Bitcoin Cash. If you have experience as a web developer you can use BitcoinToken to create a smart contract based application. You can use a pre-defined library and launch a token in a few lines of Javascript. Or you can code your own smart contracts in Javascript.

We are in prototype stage and plan to launch towards the end of the year.

Easy to use, cheap

Easy to use

BitcoinToken allows you to build Bitcoin and smart contract enabled apps without prior knowledge of Bitcoin. For example, you can issue your own token in a few lines of Javascript.

We currently support ERC20-style fungible tokens. Support for non-fungible tokens will be added soon. Eventually you will be able to program your own smart contracts using BitcoinToken.

Cheap to operate

Ethereum is a great tool for building smart contracts, but it is relatively expensive to use. Every action in a smart contract, such as sending a token or breading a kitten, costs one Ethereum transaction fee. This fee hovers around 10 cents (see red line in chart).

Fees on Bitcoin Cash are mostly a fraction of a cent (see green line in chart). BitcoinToken compiles smart contracts into Bitcoin Cash. Therefore actions in a smart contract build with BitcoinToken only cost a Bitcoin Cash transaction fee, which is often 100x below the price for an Ethereum transaction fee.

Unbounded computational scalibility

Smart contracts on BitcoinToken work differently from smart contracts on Ehtereum. In Ethereum every miner runs a computation for every action in a smart contract. The result of the computation is the new state that gets stored on the Ethereum Blockchain.

BitcionToken shifts some of the work from the miners to the users. In particular, users compute the new state and send only the new state to the miners. Miners only do minimal computational work. This is more scalable because there are usually only 2 or 3 users involved in a smart contract action, as opposed to thousands of users.

This makes it possible to build smart contracts that could have never been built before.
